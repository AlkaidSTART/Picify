## 一、设计总纲

### 1.1 核心气质

**白黑为骨，马卡龙蓝为魂，液态玻璃为肤** — 简约但不寡淡，高级但不疏离。

| 要素 | 规范 |
|------|------|
| 主体色 | 纯白 `#FFFFFF`（大面积铺底，不使用任何灰底） |
| 文本色 | 墨黑 `#121417`（标题）+ 墨蓝 `#2C3E5A`（正文辅助，不用灰色） |
| 点缀色 | 马卡龙蓝 `#7FB0EA`（CTA、选中态、关键标识） |
| 禁止色 | 任何灰色系（`#f7f8fa`、`#9ca3af`、`#6b7280`、灰色边框等全部移除） |
| 液态玻璃 | 见下方 §1.4 详细规范 |
| 动效 | GSAP 驱动，进场淡入上移、卡片交错滑入、数字滚动、生成进度流淌 |
| 图标 | `lucide-react`，全局禁止 Emoji |

### 1.2 配色体系

全站零灰色。蓝调渗透到边框、阴影、辅助文字，让马卡龙蓝不止出现在按钮上。

```
--color-base: #FFFFFF              → 纯白
--color-ink: #121417               → 主文本
--color-muted: #3A5073             → 墨蓝辅助文字（替代灰色）
--color-brand: #E8F2FF             → 马卡龙蓝极浅底
--color-brand-strong: #7FB0EA      → 马卡龙蓝
--color-brand-deep: #5A93D1        → hover 与强化态
--color-accent-glow: rgba(127,176,234,0.25)  → 光晕与渐变起点
--color-panel: rgba(255,255,255,0.58)         → 玻璃面板（偏透，让底层光晕可见）
--color-border: rgba(127,176,234,0.22)        → 蓝调边框
--color-shadow: rgba(127,176,234,0.12)        → 蓝光阴影
```

### 1.3 布局核心原则

**一看就懂：用户进门不需要思考"从哪开始"。**

- 首页只做三件事：展示结果冲击力 → 给出角色入口 → 轻量收尾
- Dashboard 一次只呈现当前步骤，做完自动推进，路径唯一

### 1.4 液态玻璃高级感规范

液态玻璃是让用户感知"高级"的核心手段。不是简单加 `backdrop-filter: blur`，而是通过**多层次叠加**制造真实的液态折射感。

#### 1.4.1 三层叠加结构

每个玻璃面板由三层叠加构成，这才是"液态"的关键：

```
┌─ 高光层（Top Highlight）────────────────────────┐
│  顶部 1px 高光线 + 30% 白色渐变内反射           │
│  → 模拟光线从上方射入液面时的顶部折射            │
│  → 这是视觉上判断"玻璃感"的第一信号              │
├─ 玻璃体（Glass Body）───────────────────────────┤
│  半透明白色底 + backdrop-blur                    │
│  → 模拟液态材质的身体透光性                      │
│  → 透明度不能太高（看穿）也不能太低（不透）       │
├─ 底部折射层（Bottom Refraction）────────────────┤
│  底部 1px 蓝色调高光线 + 底部柔和蓝色阴影        │
│  → 模拟光线穿透液体后在底部形成的色散             │
│  → 这是和普通毛玻璃（只有模糊）的核心差异点       │
└─────────────────────────────────────────────────┘
```

#### 1.4.2 三级玻璃参数

| 等级 | 场景 | blur | 透明度 | 顶部高光 | 底部折射 | 描边 | 阴影 |
|------|------|------|--------|----------|----------|------|------|
| 浅玻璃 | 装饰卡片、分组区块 | 12px | 0.52 | 0.3 白渐变 | 蓝调 0.08 | 蓝 0.16 | 蓝光柔 |
| 标准玻璃 | 主要内容面板 | 18px | 0.65 | 0.4 白渐变 + 1px 白线 | 蓝调 0.12 + 1px 蓝线 | 蓝 0.22 | 蓝光中 |
| 深玻璃 | Header、吸底栏 | 24px | 0.82 | 0.5 白渐变 + 1px 白线 | 蓝调 0.15 + 1px 蓝线 | 蓝 0.28 | 蓝光深 |

#### 1.4.4 液态玻璃的关键 — 背景必须"有东西可透"

玻璃感的前提是**背景有色彩和层次可以透过面板折射出来**。纯白背景上的玻璃面板看不出来。因此：

- 页面底层必须铺设**蓝色渐变光晕 blob**，玻璃面板才能呈现折射效果
- 每张卡片背后的微妙蓝光晕是"液态感"的核心来源
- 选中态卡片额外叠加一圈 `box-shadow: 0 0 24px rgba(127,176,234,0.25)` 蓝色外发光，突出选中状态

#### 1.4.5 其他高级感细节

| 细节 | 实现 | 作用 |
|------|------|------|
| 卡片 hover 微浮 | `transform: translateY(-2px)` + 阴影加深 | 实体感，不是扁平贴片 |
| 选中态蓝色外晕 | `box-shadow: 0 0 24px rgba(127,176,234,0.25)` | 液态边界发光 |
| 按钮 hover 光晕扩散 | `box-shadow: 0 0 0 6px rgba(127,176,234,0.15)` | 呼吸脉冲感 |
| 圆角统一 | `rounded-2xl`（12px）卡片 / `rounded-xl`（8px）内部元素 | 液态边缘无锐角 |
| 输入框聚焦 | 蓝色边框渐显 + 极淡蓝色底 `bg-[var(--color-brand)]` | 聚焦也有液态反馈 |
| 分隔线 | 蓝调渐变 `linear-gradient(90deg, transparent, rgba(127,176,234,0.2), transparent)` | 不是硬切线，是液面分界 |

## 二、页面级方案

### 2.1 首页（Landing）— 三层冲击

**目标**: 3 秒内"看到结果 → 知道我能用 → 点进去"。

#### 层次一：视觉冲击层（Hero）

```
┌──────────────────────────────────────────────────┐
│  深玻璃 Header: Logo │ 进入创作台                │
├──────────────────────────────────────────────────┤
│                                                  │
│  不写提示词，                                    │
│  直接做出可发布视觉                               │
│                                                  │
│  从角色到参数到生成，收敛在一个流程里              │
│                                                  │
│  [免费开始]  [查看适用角色]                       │
│                                                  │
│  ┌──────────────────────────────────┐            │
│  │                                  │            │
│  │  ← 真实样例图左右滑动切换 →      │            │
│  │     4 张不同场景的生成结果        │            │
│  │     底部标注场景名               │            │
│  │     (液态玻璃容器包裹)           │            │
│  │                                  │            │
│  └──────────────────────────────────┘            │
│                                                  │
│  背景蓝色光晕 blob 漂浮流动                      │
│                                                  │
└──────────────────────────────────────────────────┘
```

- **真实样例图代替模拟界面**：直接展示 4 张高质量生成结果，用户一眼理解"我能得到这种图"
- 样例图容器用标准玻璃包裹，底部标注场景名（如"电商白底主图""小红书封面"）
- 蓝色光晕 blob 缓慢漂浮在 Hero 底层，透过玻璃面板折射出液态感
- GSAP：标题逐行 stagger 淡入 → CTA 上浮 → 样例图 scale 揭晓 → blob 持续流动

#### 层次二：角色入口层

```
┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐
│ 商店  │  │ 钢笔  │  │ 公文包│  │ 消息  │
│ 图标  │  │ 图标  │  │ 图标  │  │ 图标  │
│      │  │      │  │      │  │      │
│ 电商  │  │ 自媒体│  │ 设计师│  │ 职场  │
│ 商家  │  │  人   │  │      │  │ 白领  │
│      │  │      │  │      │  │      │
│ 商品主│  │ 平台封│  │ 灵感草│  │ 汇报插│
│ 图... │  │ 面... │  │ 图... │  │ 图... │
│      │  │      │  │      │  │      │
│进入 → │  │进入 → │  │进入 → │  │进入 → │
└──────┘  └──────┘  └──────┘  └──────┘
```

- 4 张浅玻璃卡片横排，图标 + 名称 + 一句业务描述 + "进入场景" 蓝色小按钮
- hover：卡片微浮 + 蓝色外晕 + 图标 scale(1.1)
- GSAP：ScrollTrigger stagger 入场，间隔 0.12s
- 背景同样有蓝色 blob，让卡片有东西可以折射

#### 层次三：一句话收尾

```
基础模式 1 次 / 张   ·   高级模式 2 次 / 张   ·   游客可免费体验 1 次
```

- 居中一行文字，不占整块区域
- "免费体验"用马卡龙蓝色

#### 首页不包含的内容

- ❌ 四步流程说明（Dashboard 承担）
- ❌ 模式对比详情（Dashboard 步骤 3 承担）
- ❌ 套餐价格（收尾行一句话即可，不展开）
- ❌ FAQ（不需要）

#### Hero 背景光晕系统

液态玻璃的前提 — 背景必须有东西可折射：

```
blob1: 左上方 8% 5%, radius 450px, rgba(127,176,234,0.18)
blob2: 右下方 80% 55%, radius 550px, rgba(127,176,234,0.10)
blob3: 中偏左 35% 40%, radius 300px, rgba(127,176,234,0.06)

GSAP: 三个 blob 以不同速度做圆周/八字形位移
  blob1: 25s 一周期
  blob2: 30s 一周期
  blob3: 20s 一周期
→ 三个光晕永远不完全重叠，产生持续的液态折射变化
```

角色入口层下方也铺设一个较淡的 blob，让该区域的玻璃卡片同样有折射效果。

### 2.2 Dashboard（创作台）— 单栏步骤式引导

**核心原则：一次只做一件事，做完自动推进。用户不需要思考"从哪开始"。**

#### 整体布局

```
┌──────────────────────────────────────────────────────┐
│  深玻璃 Header: Logo │ ①──②──③──④ │ 余额 │ 返回   │
│                     粗进度条，当前步蓝色实色填充        │
├──────────────────────────────────────────────────────┤
│                                                      │
│    当前步骤标题（大字，32-40px）                       │
│    当前步骤说明（一行墨蓝小字）                        │
│                                                      │
│    ┌────────────────────────────────────────┐        │
│    │                                        │        │
│    │    步骤内容区域（居中，max-w-3xl）       │        │
│    │    占满宽度，有留白呼吸感                │        │
│    │                                        │        │
│    └────────────────────────────────────────┘        │
│                                                      │
│    ┌────────────────┐                                │
│    │  继续下一步 →   │  ← 蓝色大 CTA（选中后自动出现） │
│    └────────────────┘                                │
│                                                      │
└──────────────────────────────────────────────────────┘
```

#### A. 深玻璃 Header + 粗步骤进度条

不再是四个小圆点，改为**横贯顶部的粗进度条**：

```
  ① 选人群      ② 选场景      ③ 填参数      ④ 看结果
  ████████       ░░░░░░░       ░░░░░░░       ░░░░░░░
  (蓝色实填)    (浅蓝底)      (浅蓝底)      (浅蓝底)

当前步骤蓝色实色填充，已完成步骤深蓝实色，未来步骤白底+蓝调边框
秒懂"我在第几步"
```

- 高度 48px，圆角条形
- GSAP：步骤切换时，蓝色填充从左到右"流淌"动画（0.4s）
- 步骤标签可点击回退（已完成的步骤）

满屏展示四类人群，居中、松散：

```
└────────────┘  └────────────┘

┌────────────┐  ┌────────────┐
│  公文包图标  │  │  消息图标   │
│            │  │            │
│   设计师    │  │  职场白领   │
│  灵感草图、  │  │  汇报插图、  │
│  品牌概念   │  │  课件素材   │
│  4 个场景  │  │  3 个场景   │
└────────────┘  └────────────┘
```

- 2×2 网格，标准玻璃卡片，宽松 padding
- 选中态：蓝色外晕 + 顶部蓝色渐变高光条 + `scale(1.02)`
- GSAP：选中后卡片缩放弹跳 + "继续下一步"CTA 从底部上浮出现

#### C. 步骤 ② — 选场景

选中人群后，场景列表从右侧滑入：

```
┌──────────┐ ┌──────────┐ ┌──────────┐
│ 缩略图   │ │ 缩略图   │ │ 缩略图   │
│ placeholder│ │ placeholder│ │ placeholder│
├──────────┤ ├──────────┤ ├──────────┤
│ 商品主图  │ │ 促销海报  │ │ 详情配图  │
│ 白底/场景 │ │ 限时/满减 │ │ 卖点/对比 │
└──────────┘ └──────────┘ └──────────┘
```

- 3-4 列网格，浅玻璃卡片
- GSAP：stagger 从右侧滑入，选中后自动推进到步骤 3

#### D. 步骤 ③ — 填参数 + 选模式 + 生成

居中表单，max-w-2xl，不贴边：

```
┌──────────────────────────────────┐
│  参数表单（液态标准玻璃面板）       │
│  每组 ≤4 项，超出折叠              │
│  分隔线用蓝调渐变                  │
│  输入框聚焦 → 蓝色边框 + 淡蓝底    │
│                                  │
│  ── 模式选择 ──────────────────  │
│  [基础模式]     [高级模式 ★推荐]   │
│  1次/张         2次/张            │
│  快速出图       AI深度推理+更高质量 │
│                                  │
│  ┌────────────────────────────┐  │
│  │     立即生成（1 次）        │  │
│  └────────────────────────────┘  │
│  剩余 45 次                     │
└──────────────────────────────────┘
```

- 模式切换用两张并排卡片，高级模式加蓝色光晕边框 + "推荐" 角标
- 生成按钮：马卡龙蓝大圆角 `h-12`，hover 蓝色光晕扩散
- GSAP：按钮点击弹跳 `scale(0.95)→1.05→1`，生成中进度条蓝色渐变流淌

#### E. 步骤 ④ — 看结果

结果全屏居中，图占比大，不给其他信息抢注意力：

```
┌──────────────────────────────────────┐
│                                      │
│     ┌────────────────────────┐       │
│     │                        │       │
│     │   生成的图片（大图）     │       │
│     │   液态玻璃容器包裹      │       │
│     │                        │       │
│     └────────────────────────┘       │
│                                      │
│     [下载原图]   [重新生成]           │
│                                      │
│     🧠 AI 设计师的思考过程 ▼         │
│     (可折叠，GSAP 展开动画)          │
│                                      │
│     CDK 兑换（折叠，低优先级）        │
│                                      │
└──────────────────────────────────────┘
```

- GSAP 结果揭露：`blur(20px) → blur(0) + scale(0.95→1)`，6s
- 推理过程展开：GSAP height 动画 + 内容 stagger 淡入
- 下载：蓝底白字 / 重试：白底蓝字描边

#### F. 自动推进交互

- 选完人群 → 场景列表自动从右侧滑入（0.4s GSAP），无需手动点"下一步"
- 选完场景 → 参数表单自动展开（0.3s GSAP）
- 顶部进度条始终可点击回退到已完成步骤
- 步骤 ③ 不自动推进（需要用户主动点生成）

#### G. Dashboard 背景光晕

与首页类似但更克制：

```
一个居中偏右上方的蓝色 blob，radius ~500px，rgba(127,176,234,0.08)
缓慢位移，20s 周期
→ 各步骤的玻璃面板折射出微妙的液态感

```

## 三、GSAP 动效体系

### 3.1 安装与引入

```
npm install gsap
```

客户端组件中通过 `"use client"` + `useGSAP`（`gsap/react`）管理生命周期。

### 3.2 动效清单

| 编号 | 位置 | 动效 | 时长 | 触发 | GSAP API |
|------|------|------|------|------|----------|
| A1 | Header | 从顶部淡入下滑 | 0.5s | 页面加载 | `gsap.from` |
| A2 | Hero 标题 | 逐行 stagger 淡入上移 | 0.6s | 页面加载 | `gsap.from` stagger |
| A3 | Hero 样例图 | scale(0.95→1) + fadeIn | 0.5s | 页面加载 delay 0.3s | `gsap.from` |
| A4 | 背景 blob | 三组缓慢圆周位移 | 20-30s 循环 | 始终 | `gsap.to` repeat:-1 |
| A5 | 人群卡片 | stagger 淡入上移 | 0.4s/项 | ScrollTrigger | `gsap.from` stagger |
| B1 | 步骤进度条 | 蓝色填充流淌 | 0.4s | 步骤切换 | `gsap.to` width |
| B2 | 步骤内容 | 从右侧滑入 | 0.4s | 自动推进 | `gsap.from` x: 80 |
| B3 | 继续按钮 | 从底部上浮 | 0.3s | 选中完成后 | `gsap.from` y: 20 |
| B4 | 人群选中 | scale 弹跳 + 蓝色外晕 | 0.4s | 点击选中 | `gsap.timeline` |
| B5 | 模式切换 | scale + 边框蓝色渐显 | 0.3s | 切换 | `gsap.to` |
| B6 | 生成按钮 | 弹跳缩放 | 0.4s | 点击 | `gsap.timeline` |
| B7 | 生成进度条 | 蓝色渐变流淌 | 持续 | 生成中 | `gsap.to` xPercent |
| B8 | 结果图片 | blur→0 + scale | 0.6s | 生成完成 | `gsap.from` |
| B9 | 推理展开 | height + stagger | 0.4s | 点击展开 | `gsap.to` + stagger |
| C1 | 移动端吸底栏 | 方向敏感 show/hide | 0.3s | 滚动 | ScrollTrigger toggle |

### 3.3 动效原则

- **不遮挡信息**：动效期间所有文字保持可读
- **不干扰操作**：动画不阻塞点击、不锁屏
- **可访问性**：`prefers-reduced-motion: reduce` 时所有动效降级为瞬间切换
- **性能**：仅对 `transform` 和 `opacity` 做动画，不触发 layout/paint

## 七、风险与约束

1. **可读性优先**：液态玻璃透明度必须保证文字 WCAG AA 对比度（4.5:1），不因"好看"牺牲可读
2. **零灰色硬约束**：全站禁止灰色。辅助文字墨蓝 `#3A5073`，边框/阴影蓝调 rgba，背景纯白。`gray-*`、`slate-*` 不可出现
3. **液态玻璃依赖背景光晕**：如果 blob 没渲染，玻璃面板会显得"只是半透明"而非"液态"。blob 是必须项不是可选项
4. **GSAP 许可**：标准版免费用于非商业，商用需 Business 许可 — 需确认项目授权
5. **组件粒度**：不为拆分而拆分，简单展示内容可内联
6. **移动端首屏**：Hero 标题 + CTA 必须在首屏完整可见

## 八、视觉参考

### 液态玻璃高级感

- **Apple visionOS** — 系统面板的三层结构：顶部高光线 + 半透体 + 底部色散。这是"液态"和"毛玻璃"的核心差异
- **Stripe** — 背景渐变 blob 让玻璃面板有东西可折射，没有 blob 的玻璃只是"磨砂"
- **Linear** — 简约色调 + 精准微动效，克制的高级

### 马卡龙蓝

- `#7FB0EA` — 柔和不刺眼，搭配纯白时呼吸感强
- 渗透到边框、阴影、辅助文字、背景光晕，形成完整蓝调体系而非孤立的蓝色按钮
